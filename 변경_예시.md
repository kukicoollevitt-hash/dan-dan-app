# 정답 재분배 변경 예시

## 예시 1: people1_06 (이순신) - 정답을 ①번으로 변경

### 변경 전
```javascript
q1_opts: ['① 어린진', '② 장사진', '③ 학익진', '④ 원진'],
answerKey: {
  q1: '3',  // 정답: ③ 학익진
}
```

### 변경 후
```javascript
q1_opts: ['① 학익진', '② 어린진', '③ 장사진', '④ 원진'],
answerKey: {
  q1: '1',  // 정답: ① 학익진 (내용 동일, 위치만 변경)
}
```

**변경 사항**:
- ③ 학익진 → ① 학익진 (정답이 3번에서 1번으로 이동)
- 나머지 오답들도 순서 재배치

## 핵심 원칙

### ✅ 유지되는 것
1. **정답의 실제 내용** (텍스트)
2. **문제의 의도와 난이도**
3. **오답 선택지들의 내용**
4. **설명(explain)의 내용**

### 🔄 변경되는 것
1. **보기들의 순서** (재배치)
2. **보기 번호** (①②③④ 재부여)
3. **answerKey의 숫자** (새로운 정답 위치)

### 🎯 목적
- 정답이 특정 번호에 편중되지 않도록 균등 분배
- 학생들이 패턴이 아닌 내용을 이해하여 문제 풀이
- 학습 품질 향상 및 공정성 확보

## 검증 방법

각 변경사항은 다음과 같이 검증됩니다:

1. **정답 내용 확인**: 변경 전후 정답 텍스트가 동일한지 확인
2. **정답 위치 확인**: answerKey의 숫자가 올바른 위치를 가리키는지 확인
3. **보기 번호 확인**: 각 보기의 번호(①②③④)가 순서대로 부여되었는지 확인
4. **분포 확인**: 전체적으로 정답이 균등하게 분배되었는지 확인

모든 검증이 통과되었습니다! ✅
